# VM Configuration Requirements — SUPFile Infrastructure

**Document Version**: 1.0
**Project**: SUPFile — Multi-Site Cloud Storage Infrastructure
**Scope**: Virtual machine specifications for all three data centers (NY, Paris, Toronto)
**Last Updated**: April 2026

---

## 1. Overview

This document defines every virtual machine required to run the SUPFile infrastructure across all three data centers. The POC uses virtualization (Proxmox VE or VMware ESXi) on physical hosts. Each VM has defined CPU, RAM, disk, and network interface requirements.

### Hypervisor Hosts (Physical Servers)

Each primary DC (NY, Paris) requires a minimum of **3 physical hypervisor hosts** to support the VM workload with N+1 redundancy. Toronto requires **2 physical hosts** (reduced workload).

| Physical Host Role | CPU | RAM | Disk | Qty per Primary DC | Qty Toronto |
|-------------------|-----|-----|------|--------------------|-------------|
| Compute Host (App/DB/Mgmt VMs) | 2× Xeon 16-core (32 cores) | 256 GB ECC | 2× 1TB NVMe (RAID 1, OS + VM storage) | 2 | 1 |
| Storage Host (SAN VMs) | 2× Xeon 8-core (16 cores) | 128 GB ECC | 12× 4TB HDD (RAID 6) + 2× 500GB NVMe (cache) | 3 | 1 |
| **Total physical servers** | | | | **5 per primary DC** | **2 for Toronto** |

## 6. Operating System Standardization

| Component | OS | Justification |
|-----------|-----|---------------|
| All Linux VMs | Debian 12 (Bookworm) or Ubuntu 24.04 LTS | LTS support (5+ years), wide package availability, strong community, free |
| Firewalls | OPNsense 24.x | Open-source, FreeBSD-based, CARP HA, built-in IPsec, web GUI |
| Hypervisor | Proxmox VE 8.x | Open-source, KVM-based, built-in HA clustering, web GUI, free tier |

### Standard Packages on All Linux VMs

| Package | Purpose |
|---------|---------|
| wazuh-agent | Ship security logs to SIEM |
| zabbix-agent2 | Ship metrics to monitoring |
| fail2ban | IP ban on SSH brute force |
| ufw / nftables | Host-level firewall |
| unattended-upgrades | Automatic security patches |
| chrony | NTP time sync (critical for log correlation) |
| auditd | Kernel-level audit logging |
